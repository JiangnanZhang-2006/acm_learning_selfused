# 插头 DP

插头 DP（轮廓线 DP）是一种处理网格图路径、连通性等问题的 DP 方法。

## 基本概念

插头 DP 使用轮廓线表示当前已处理的格子和未处理格子的分界线，用括号表示法或最小表示法表示连通性。

## 基本框架

```cpp
// 问题：哈密顿回路计数（示例）
// 状态：使用轮廓线表示当前已处理的格子和未处理格子的分界线
//       用括号表示法或最小表示法表示连通性

// 插头 DP 通常使用哈希表存储状态
// 状态表示：当前格子位置 + 轮廓线上的插头状态
// 转移：根据当前格子的选择（是否放置路径）更新插头状态

// 这是一个复杂的 DP 类型，通常需要：
// 1. 状态压缩（用数字表示轮廓线状态）
// 2. 状态转移（合并、新建、延续插头）
// 3. 滚动数组优化

// 具体实现较为复杂，这里给出框架
struct State {
    int mask;  // 轮廓线状态
    long long val;  // 方案数或最优值
};

unordered_map<int, long long> dp[2];  // 滚动数组

void plugDP(int n, int m) {
    int cur = 0, nxt = 1;
    dp[cur][0] = 1;  // 初始状态
    
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            dp[nxt].clear();
            for (auto& [mask, val] : dp[cur]) {
                // 根据当前格子 (i, j) 和 mask 状态进行转移
                // 更新 dp[nxt]
            }
            swap(cur, nxt);
        }
    }
}
```

## 常见应用

- 哈密顿回路/路径计数
- 网格图的连通性问题
- 棋盘覆盖问题
- 路径统计问题

## 状态表示方法

1. **括号表示法**：用左右括号表示插头的配对关系
2. **最小表示法**：用数字表示连通分量

## 注意事项

插头 DP 实现较为复杂，需要：

- 理解轮廓线的概念
- 掌握状态压缩和编码/解码
- 正确处理各种转移情况（新建、合并、延续插头）
