# VSCode C++ IntelliSense 修复步骤

## 问题：`bits/stdc++.h` 文件未找到

### 方法 1：重置 IntelliSense（推荐）

1. 按 `Cmd + Shift + P` 打开命令面板
2. 输入并执行：**C/C++: Reset IntelliSense Database**
3. 等待重新索引完成（右下角会显示进度）
4. 按 `Cmd + Shift + P`，执行：**Developer: Reload Window**

### 方法 2：手动检查配置

1. 按 `Cmd + Shift + P`
2. 输入并打开：**C/C++: Edit Configurations (UI)**
3. 在 **Include path** 部分，确保有以下路径：
   ```
   ${workspaceFolder}/**
   ${workspaceFolder}/include
   ```
4. **Compiler path** 应该是：`/usr/bin/clang++`
5. **IntelliSense mode** 应该是：`macos-clang-arm64`
6. **C++ standard** 应该是：`c++17`

### 方法 3：清理缓存并重启

在终端中执行：
```bash
cd "/Users/jiangnanzhang/Library/Mobile Documents/com~apple~CloudDocs/acm"
rm -rf .vscode/ipch
rm -rf ~/.vscode/extensions/ms-vscode.cpptools-*/bin/.browse.VC.db*
```

然后完全退出并重新打开 VSCode。

### 方法 4：重新安装 C++ 扩展

1. 打开扩展面板（`Cmd + Shift + X`）
2. 搜索 "C/C++"
3. 卸载 Microsoft 的 C/C++ 扩展
4. 重启 VSCode
5. 重新安装扩展

### 验证修复

编译测试：
```bash
cd atcoder
clang++ -std=c++17 -I../include B.cpp -o B
./B
```

如果编译成功，说明配置正确，只是 IntelliSense 缓存问题。

```cpp

```
